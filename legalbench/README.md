# LegalBench external-validation harness — Aethis Eligibility Module

Companion artefact to §6.10 of *Confidently Wrong: Exception Chain
Collapse in Frontier LLM Rule Evaluation* (Simpson, Kozak, Doake,
2026; see [`../paper/`](../paper/)).

This subdirectory contains every per-task project, source rule,
guidance file, runner, test result, and the statistical-analysis
pipeline used to produce the headline LegalBench result in §6.10:

> Across 9 LegalBench tasks (949 held-out cases), the Aethis
> Eligibility Module is significantly more accurate than each of
> three frontier LLMs by exact two-sided combined paired-binomial
> McNemar's test — *p* < 0.001 vs Claude Sonnet 4.6, *p* = 0.003 vs
> Claude Opus 4.7, *p* < 0.001 vs GPT-5.4.

See [`docs/statistical-summary.md`](docs/statistical-summary.md) for
the full per-task table.

## Methodology

- **Source audit**: each `sources/rule.md` is verbatim from the
  upstream LegalBench task description (Guha et al. 2023, CC BY 4.0).
  A grep gate forbids DSL pseudo-code, operator names, and
  field-registry sections in source files.
- **Bundle authoring**: each task's bundle is generated by the Aethis
  authoring pipeline from a small set (4–8) of statute-derived test
  cases written by the authors — *not* drawn from LegalBench fact
  patterns.
- **Two-channel SME guidance** where used: rule-authoring hints
  (server-side, affect DSL generation) and runtime extractor hints
  (loaded into the LLM extractor prompt). Both grounded in independent
  practitioner authority. See `docs/sme-guidance-channels.md`.
- **Held-out evaluation**: deterministic 50-50 dev/holdout partition
  via `tools/test_split.py --seed 7 --dev_fraction 0.5`. Hint
  iteration confined to dev half; holdout evaluated once with frozen
  hints. See `docs/heldout-methodology.md`.
- **Pre-registered random sampling** for non-curated tasks via
  `tools/random_task_pick.py --seed 42`, `--seed 43`, `--seed 44`,
  committed before authoring. See `docs/preregistration.md`.
- **Statistical analysis** via `tools/significance.py`: Wilson 95% CIs,
  exact two-sided McNemar's per task, combined paired-binomial across
  tasks, Cohen's h. Pure Python, no external stats deps beyond
  `pyyaml` + the LegalBench loader.

## Reproduction

```bash
# Run a single task's engine + held-out evaluation
uv run domains/legalbench-hearsay/run.py --eval-split holdout \
    --output domains/legalbench-hearsay/results/holdout.json

# Verify held-out partition reproducibility
uv run tools/repro_check.py

# Combined statistical analysis across all 9 tasks
uv run tools/significance.py \
    --tasks docs/significance_tasks.yaml \
    --output docs/statistical-summary.md
```

LLM baselines use the upstream LegalBench `base_prompt.txt` few-shot
prompts via `domains/_lib/few_shot_baseline.py`. Engine evaluation
calls `aethis-core` via the public REST API (see project README at the
repo root).

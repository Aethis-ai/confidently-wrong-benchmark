## Authoring + run protocol (per existing seed-42/43 protocol)

For each picked task:

1. Fetch the verbatim upstream LegalBench Task description into
   `domains/legalbench-<task>/sources/rule.md`. Grep gate must pass
   (no `Engine pattern:`, `And(`, `Or(`, `Not(`, `Field Registry`).
2. CLI-canonical project layout under `domains/legalbench-<task>/`.
3. Author bundle from 4–8 statute-derived test cases (not LegalBench
   fact patterns). `aethis_create_bundle` →
   `aethis_generate_and_test` → `aethis_publish`.
4. Vertical slice: ~10 cases. Gate: ≥9/10 correct. If no, diagnose
   before scaling.
5. Held-out methodology: `tools/test_split.py --seed 7 --dev_fraction 0.5`.
   Iterate hints (if any) on dev half only; freeze; run holdout once.
6. LLM baselines (claude-sonnet-4-6, claude-opus-4-7, gpt-5.4) on the
   same holdout indices, using the upstream LegalBench `base_prompt.txt`
   few-shot prompt. Run via `domains/_lib/few_shot_baseline.py`.
7. Add task entry to `docs/significance_tasks.yaml`.
8. Re-run `uv run tools/significance.py` to refresh the headline
   statistical table.

## Pre-commitments

- Whatever held-out accuracy and per-task McNemar's *p* lands for
  each task is reported in §6.10. No re-rolling of seeds.
- If either task surfaces a structural mismatch with the Eligibility
  Module (e.g. multi-label output, free-form numeric answers), that
  is reported as a structural-fit observation in §6.10.3, not
  hidden.
- The combined paired-binomial across all tasks (now 11) is the
  load-bearing test. Per-task ties or losses on these two tasks do
  not invalidate the combined headline; the paper text reflects
  whatever lands.
- Cost ceiling per task: ≤ $5 (authoring + extractor + LLM baseline).
  Hard cap; re-approval needed beyond.
